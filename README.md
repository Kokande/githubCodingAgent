# Code Agent для МегаШколы ИТМО
Успел реализовать и захостить только первую часть. Если бы не работа (которая тоже завязана на агентах), думаю успел бы это всё по-человечески реализовать. При создании issue создаётся pull-request с изменениями. Извиняюсь, что без звука демо - сосед спал во время записи:(
Структура проекта:
- src (корень сервиса)
  - agent (директория с агентом)
    - code_agent (папка с код-агентом)
  - cfg (конфиги)
    - secret.properties.example (пример .env файла с токенами ГХ и гигачата, для запуска .example убрать)
    - service.properties (там должны были быть общие настройки)
  - config.py (файл с настройками)
  - main.py (файл с сервисом)
- докерфайлы и т.д.

## Архитектура решения
Сервис на фастапи, принимающий сообщения по эндпоинту /webhook
Внутри парсится создающиеся Issues и кидаются в агента. Сам агент представлен в виде ReAct агента с тулами: просмотр файловой структуры, чтение файлов, апдейт файлов. По окончанию ReAct ноды идёт создание PR.

## Описание
Репо с которым работает агент: https://github.com/Kokande/geoProjFastAPI
Взял маленький репо - не успел для большого подогнать всё: гигачат бывает падает на 500 и таймаутах.
Можно было бы прикрутить память по-круче, чтобы ему не пришлось тул-колы плодить чисто чтобы вспомнить репо.
Использован гигачат тк на него куплены токены, а с другими разобраться не успел.
